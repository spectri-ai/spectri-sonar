# RFC-YYYY-MM-DD: [Descriptive Title]

> **IMPORTANT**: This is a Request for Comments (RFC) — a pre-decision exploration document. Nothing here represents confirmed direction until it transitions to "Accepted" status and generates corresponding ADRs.
>
> **What an RFC is NOT**: An RFC is not an implementation guide, not a procedure, and not a single-solution proposal. If you have only one approach, you haven't explored enough — add at least one alternative. Document type hierarchy: **RFC** (explore options) → **ADR** (record decision) → **Spec** (define feature) → **Plan** (implement).
>
> **Naming**: RFCs use date-based naming (`RFC-YYYY-MM-DD-slug.md`) to maintain chronological context and avoid confusion with ADR sequential numbering. The slug ensures uniqueness for same-day RFCs.
>
> **Evolution**: RFCs evolve through dated discussion sections. Never delete previous content — add new dated sections at the bottom to show how thinking evolved. This preserves context and prevents accidental loss of important reasoning.
